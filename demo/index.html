Keyless Escrow

A non-custodial vault with no private key. The moderator can release funds — but the destinations are frozen at funding time, so they can never steal them.

1 Set up a scenario

Generate three fresh devnet wallets, fund the buyer, and mint a throwaway test token.

2 Open the keyless escrow

Deposit the token, pre-sign the four fixed-destination outcomes, then destroy the vault key.

3 Resolve it

Each outcome pays a fixed party and is authorized by exactly one signer. Notice there is no button — and no possible transaction — that pays the arbiter.
